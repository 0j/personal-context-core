EXACT FIELD NAMES — SPELLING MUST MATCH EXACTLY
════════════════════════════════════════════════════════════════
The output is validated against a strict JSON schema. One wrong key name fails the
entire response. Use the field names below and no others.

  entities_detected items — use EXACTLY:
    "entity_type"      ← NOT "type", NOT "category", NOT "kind", NOT "entityType"
    "canonical_name"   ← NOT "name", NOT "label", NOT "title", NOT "canonicalName"
    "summary"          ← required string, even if short
    "confidence"       ← float in [0.0, 1.0]
    "source_refs"      ← array of source-ref objects (see below)

  relationships_detected items — use EXACTLY:
    "from_name"        ← NOT "entity_a", NOT "subject", NOT "source", NOT "fromName"
    "to_name"          ← NOT "entity_b", NOT "object", NOT "target", NOT "toName"
    "relation_type"    ← NOT "relationship_type", NOT "type", NOT "relationType"
    "confidence"       ← float in [0.0, 1.0]
    "source_refs"      ← array of source-ref objects

  memory_candidates items — use EXACTLY:
    "candidate_type"       ← one of: fact | preference | goal | relationship | other
    "memory_type"          ← one of: fact | preference | goal | belief | skill | experience | relationship | routine
    "statement"            ← complete declarative sentence about the user
    "summary"              ← ≤10-word phrase
    "temporal_scope"       ← one of: permanent | long_term | short_term | session
    "sensitivity"          ← one of: public | internal | private | highly_private
    "confidence"           ← float in [0.0, 1.0]
    "salience_score"       ← float in [0.0, 1.0]
    "related_entity_names" ← array of strings ([] if none)
    "source_refs"          ← array of source-ref objects — MUST NOT be empty
    "rationale"            ← one sentence explaining why this is memorable

  contradictions_found items — use EXACTLY:
    "existing_memory_id"  ← id from the approved-memories list above
    "new_claim"           ← the conflicting statement from this conversation
    "confidence"          ← float in [0.0, 1.0]
    "source_refs"         ← array of source-ref objects

  stale_candidates items — use EXACTLY:
    "memory_id"   ← id from the approved-memories list above
    "reason"      ← short explanation string
    "confidence"  ← float in [0.0, 1.0]

  source-ref objects — use EXACTLY:
    "message_id"  ← id of the message in the conversation above (required)
    "excerpt"     ← short verbatim quote (optional)

  Top-level keys — all six are required, use empty arrays when there is nothing to report:
    "entities_detected"
    "relationships_detected"
    "memory_candidates"
    "contradictions_found"
    "stale_candidates"
    "confidence_notes"      ← array of plain strings (caveats, low-confidence notes)
